# 数据标注说明

## PDF 选择标准

- 至少 2 份，最好 3 份。
- 必须包含正文、表格、图或图表。
- 优先选择能提出客观问题的文档，例如论文、实验报告、课程资料、项目说明书。

## 问题类型配比

建议 20-30 个问题：

- 文本事实题：6-8 个。
- 表格问答题：4-6 个。
- 图表理解题：4-6 个。
- 图文一致性题：3-5 个。
- 证据定位题：3-5 个。

## questions.csv 字段

```text
question_id,doc_id,question,answer,question_type,gold_node_ids,gold_pages,gold_modalities,evidence_note
```

填写规则：

- `question_id` 使用 `Q001`、`Q002`。
- `doc_id` 使用 PDF 文件名去掉扩展名后的规范化名称，脚本会把空格替换为下划线。
- `gold_node_ids` 可以有多个，用分号分隔。
- `gold_pages` 可以有多个，用分号分隔。
- `gold_modalities` 使用 `text/table/figure/caption/page`。
- `evidence_note` 写清“第几页、哪个表/图/段落”。

## 自动解析缺失时

如果脚本没有抽出关键表格或图表，把证据补到 `data/manual_nodes.csv`：

```text
node_id,doc_id,page,node_type,content,source_ref
TB_doc1_5_1,doc1,5,table,表 2 中方法 B 的 Accuracy 最高,第 5 页 表 2
```

补完后重新运行：

```bash
python scripts/06_run_pipeline.py
```
