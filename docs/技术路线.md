# 技术路线

## 总体流程

```text
复杂 PDF
  -> RAG-Anything 解析或简化 PDF 解析
  -> 节点表 nodes.jsonl
  -> 边表 edges.jsonl
  -> 问题输入 questions.csv
  -> 初步召回 Top-K candidates.csv
  -> G0/G1/G2/G3/G4 重排序 reranked.csv
  -> 指标统计 summary_metrics.csv
  -> Demo 展示答案、页码、证据关系
```

## 节点类型

- `text`：正文段落。
- `table`：表格或人工整理的表格文本。
- `figure`：图、图表或图表摘要。
- `caption`：图注或表注。
- `page`：页面结构节点。

## 边类型

- `belongs_to_page`：节点属于某一页。
- `same_page`：同页节点关系。
- `figure_caption`：图表和图注关系。
- `table_caption`：表格和表注关系。
- `text_ref_figure` / `text_ref_table`：正文引用图表或表格。

## 排序方法

- G0：初步检索原始顺序。
- G1：语义相似度排序。
- G2：`0.7 * Sim + 0.3 * PPR`。
- G3：`0.6 * Sim + 0.25 * PPR + 0.15 * Bridge`，并加入显式图表引用信号。
- G4：在 G3 基础上融合视觉 grounding、证据链连贯性和领域意图信号。

```text
G4 = G3
   + VisualWeight(q) * Visual(v, q)
   + ChainWeight(q) * Chain(v, q)
   + DomainWeight(q) * DomainIntent(v, q)
```

其中 `DomainIntent` 当前主要服务于售后知识库，识别退换货、发票、物流破损、保修维修、故障排查、安装使用、规格配件和安全警告等问题。

## Bridge 分数

```text
Bridge*(v | q) = TypeWeight(v) * DegreeNorm(v) * CrossModalRel(v | q)
DegreeNorm(v) = 1 / log(2 + deg(v))
```

解释口径：Bridge 不是简单奖励“连接多”的节点，而是奖励那些连接了多种查询相关模态的节点，同时通过类型权重和度数归一化压低页面、标题等结构节点的虚高分。
