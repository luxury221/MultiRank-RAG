# Demo 演示脚本

建议时长 5 分钟。

## 1. 选择文档和问题

现场打开 Demo，选择预置 PDF 和一个表格题或图文一致性题。

讲法：本系统面向图文表混排 PDF，目标不是重新训练模型，而是在初步检索后提升关键证据排序。

## 2. 展示 G0 Top-K

讲法：G0 是原始召回结果，可以看到候选证据中有相关内容，但关键表格或图表不一定排在最前。

## 3. 展示 G3 Top-K

讲法：G3 同时使用语义相似度、查询感知 PPR 和 Bridge 分数。PPR 让查询相关的局部图结构参与排序，Bridge 奖励连接多种相关模态的证据节点。

## 4. 展示答案和引用

讲法：系统给出答案，并展示证据节点、节点类型和页码。这里强调可追溯性。

## 5. 展示节点关系

讲法：这个节点连接了页面、图注、正文解释或表格，因此不仅文本相似，而且在当前问题相关图中具有更高证据价值。

## 结尾

本课题证明，在复杂 PDF 问答中，检索后的查询感知图结构重排序能提升关键证据排序质量，尤其适合表格、图表和图文一致性问题。
